# Tally — 产品需求文档 (PRD)

> **版本**: v0.1 草稿 | **日期**: 2026-06-13 | **平台**: iOS 17+

---

## 一、产品背景与目标

### 1.1 背景

个人财务管理通常分散在多个工具中：记账 App 管日常花销、券商 App 管投资持仓、微信/备忘录记录人情借贷。切换成本高，缺乏整体财务视图。

### 1.2 产品目标

构建一款**本地优先、隐私安全**的 iOS 记账 App，将以下三类财务数据统一管理：

- 日常支出/收入的分类记录与统计
- 多平台、多币种投资持仓的汇总展示
- 借贷往来（欠钱/外债）的跟踪与提醒

**核心价值主张**：一个 App 看清净资产全貌，数据不出设备。

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 本地优先 (Local-first) | 数据默认存于设备 SwiftData，iCloud 同步为可选配置 |
| 快速记账 | 主流程 3 步内完成一笔记录，支持常用分类快捷入口 |
| 可视化驱动 | 统计图表为一等公民，与列表并列展示 |
| 多币种 | 支持 CNY / USD / HKD 等多种货币，自定义汇率 |
| 隐私与安全 | 支持 Face ID / Touch ID 应用锁，无强制联网 |

---

## 二、目标用户

**主要用户画像**：

- 有记账习惯、希望可视化消费结构的个人用户
- 同时持有多个投资平台（A股、美股、加密货币等）资产
- 有人情借贷往来（朋友借钱、信用卡欠款、亲属互借）

**次要场景**：家庭共同记账（iCloud 共享，规划项）。

## 三、模块功能需求

### 3.1 日常账本 (Daily Ledger)

#### 3.1.1 用户故事

- 作为用户，我希望快速记录一笔花费的名称、金额、分类和时间，以便追踪每天支出。
- 作为用户，我希望查看某月各分类的消费占比图表，以便了解消费结构。
- 作为用户，我希望按周/月/年/自定义时间段查看收支统计，以便做周期性复盘。
- 作为用户，我希望区分收入和支出，以便计算净结余。

#### 3.1.2 功能点

**记账 (Create / Edit)**

- 必填字段：
  - `name` 花费名称（字符串，最长 50 字符）
  - `amount` 金额（正数，精度两位小数）
  - `category` 分类（关联 Category 实体）
  - `date` 时间（默认当前时间，可修改）
  - `type` 收支类型（`.income` 收入 / `.expense` 支出）
- 可选字段：
  - `note` 备注（最长 200 字符）
  - `paymentMethod` 支付方式（现金 / 微信 / 支付宝 / 银行卡 / 信用卡 / 其他）
  - `tags` 标签（多选，自由标签）
  - `currency` 币种（默认 CNY）
- 支持编辑与删除已有记录，删除需二次确认。
- 快速记账入口（FAB 按钮）全局可达，从任意 Tab 点击均可弹出记账表单。

**分类管理 (Category)**

- 预置分类（不可删除但可隐藏）：

  | 分类 | 图标 (SF Symbol) | 颜色 |
  |------|-----------------|------|
  | 餐饮 | fork.knife | 橙色 |
  | 交通 | car.fill | 蓝色 |
  | 购物 | bag.fill | 紫色 |
  | 居住 | house.fill | 绿色 |
  | 娱乐 | gamecontroller.fill | 粉色 |
  | 医疗 | cross.fill | 红色 |
  | 教育 | book.fill | 青色 |
  | 收入 | arrow.down.circle.fill | 薄荷绿 |
  | 其他 | ellipsis.circle.fill | 灰色 |

- 支持新增自定义分类（名称 + SF Symbol + 颜色）。
- 分类支持排序（长按拖拽）。

**列表视图 (List)**

- 按日期倒序分组，每组展示日期与当日小计（支出 / 收入）。
- 顶部显示当前时间段的区间汇总：总收入、总支出、净结余。
- 支持按分类、支付方式筛选。
- 支持关键词搜索（名称 / 备注）。
- 单元格展示：分类图标、名称、金额（支出红色 / 收入绿色）、时间。

**统计视图 (Statistics)**

- 时间段选择器：**周 / 月 / 年 / 自定义区间**（默认当月）。
- 图表类型：

  | 图表 | 说明 | 维度 |
  |------|------|------|
  | 分类占比饼图 | 各分类支出占比 | 按分类 |
  | 收支趋势柱状图 | 每日/每周收入与支出对比 | 按日/周 |
  | 折线趋势图 | 净结余累积变化 | 按日 |
  | 分类排行榜 | 支出金额 Top N 分类 | 按分类 |
  | 同环比卡片 | 本期 vs 上期支出变化（金额 + 百分比） | 按周期 |

- 图表支持点击查看对应分类的明细列表。
- 空状态：当前时间段无记录时展示引导提示。

### 3.2 投资理财 (Investment)

#### 3.2.1 用户故事

- 作为用户，我希望录入不同平台的持仓，以便在一个地方看清总资产。
- 作为用户，我希望看到每个持仓的实时（或手动更新的）价格和浮动盈亏，以便评估投资表现。
- 作为用户，我希望按平台/按资产类别汇总查看，以便了解资产配置结构。

#### 3.2.2 功能点

**持仓管理 (Holding)**

- 字段：
  - `platform` 平台（关联 Platform 实体，如"富途"、"币安"、"雪球"）
  - `assetClass` 资产类别（枚举：`.crypto` / `.usStock` / `.aShare` / `.hkStock` / `.fund` / `.other`）
  - `symbol` 标的代码（如 `BTC`、`AAPL`、`600519`）
  - `name` 标的名称（如"比特币"、"苹果"、"茅台"）
  - `quantity` 持仓数量（正数）
  - `costPrice` 成本价（正数）
  - `currency` 计价币种（`USD` / `CNY` / `HKD` / ...）
  - `currentPrice` 现价（手动录入，后续由 PriceProvider 自动更新）
  - `lastUpdated` 价格最后更新时间
  - `note` 备注

- 支持新增、编辑、删除持仓。
- 单个持仓展示：标的名称 + 代码、数量、成本价、现价、市值、浮动盈亏（金额 + %）。

**平台管理 (Platform)**

- 字段：`name` 平台名称、`icon` 图标（SF Symbol 或自定义）、`currency` 默认币种。
- 预置常见平台：富途 / 老虎 / 币安 / 欧易 / 支付宝（基金）/ 雪球 / 其他。
- 支持自定义新增平台。

**汇总展示 (Portfolio Summary)**

- 按**平台**分组展示持仓列表（默认）。
- 支持切换为按**资产类别**分组。
- 顶部汇总卡片：
  - 总市值（统一换算为 CNY，基于配置汇率）
  - 总成本
  - 浮动盈亏（金额 + %）
- 图表：
  - 资产配置饼图（按平台 / 按类别切换）
  - 总市值趋势折线图（基于历史快照，手动触发保存）

**价格更新**

- v0.3 阶段：手动输入现价，手动触发刷新。
- v1.0 阶段：通过 `PriceProvider` 协议自动获取（详见 [api-integration.md](./api-integration.md)）。
- 价格更新时间戳显示在持仓卡片上，过期价格（>1小时）以警告色标注。

### 3.3 信用账本 (Credit Ledger)

#### 3.3.1 用户故事

- 作为用户，我希望记录借给朋友的钱和借来的钱，以便知道净债权/债务状况。
- 作为用户，我希望记录还款进度，以便跟踪每笔借贷的剩余金额。
- 作为用户，我希望设置到期提醒，以便不忘还款或催款。

#### 3.3.2 功能点

**借贷记录 (Debt)**

- 字段：
  - `counterparty` 对手方（关联 Counterparty 实体）
  - `direction` 方向（`.iOwe` 我欠 / `.owedToMe` 别人欠我）
  - `amount` 借贷总金额
  - `currency` 币种
  - `date` 发生日期
  - `dueDate` 到期日（可选）
  - `interest` 利息（可选，年利率 %）
  - `status` 状态（`.open` 未结清 / `.partial` 部分还清 / `.settled` 已结清）
  - `note` 备注
  - `repayments` 还款记录列表（关联 Repayment 实体）

**对手方管理 (Counterparty)**

- 字段：`name` 姓名、`contactInfo` 联系方式（可选）、`avatarColor` 头像颜色（用于无图片时显示缩写）。
- 支持从系统通讯录导入（规划项，需用户授权）。

**还款记录 (Repayment)**

- 字段：`amount` 还款金额、`date` 还款日期、`note` 备注。
- 每次还款自动更新 `Debt.status`（剩余金额 = 0 → `settled`）。

**列表视图**

- 分为两组：**外债（别人欠我）** / **我欠（我欠别人）**。
- 顶部汇总卡片：应收总额、应付总额、净债权/债务。
- 单元格展示：对手方头像 + 姓名、金额、到期日（若存在）、状态标签。
- 已结清的记录默认折叠（可展开查看历史）。

**详情页**

- 展示借贷基本信息 + 还款时间线（Timeline）。
- 支持"记录一笔还款"快捷入口。
- 到期日提醒设置（本地通知）。

## 四、非功能需求

### 4.1 隐私与安全

- 支持 **Face ID / Touch ID** 应用锁（设置中开启），锁定时不展示任何数据。
- 数据默认存储于设备本地，**不上传任何服务器**。
- iCloud 同步为可选开关，关闭后数据完全离线。
- 不收集用户行为数据，无第三方 Analytics SDK（除用户主动开启的可选功能）。

### 4.2 多币种

- 所有货币金额存储时保留原始币种 + 金额。
- 汇总展示时按用户配置的基础货币（默认 CNY）换算。
- 汇率手动配置，可在设置中更新（后续规划自动拉取汇率 API）。

### 4.3 性能

- 冷启动 ≤ 1.5 秒（iPhone 12 及以上）。
- 图表渲染 ≤ 500ms（数据量 ≤ 5000 条记录）。
- SwiftData 查询利用 `@Query` + `Predicate` 避免全表扫描。

### 4.4 可访问性 (Accessibility)

- 支持动态字体 (Dynamic Type)。
- 所有图标 + 图表提供 VoiceOver 标签。
- 颜色区分不作为唯一信息传达手段（配合图标/文字）。

### 4.5 暗黑模式

- 完全支持 iOS 暗黑模式，使用系统语义色（`.background`、`.secondaryBackground`、`.label` 等）。

### 4.6 离线可用

- 全部核心功能（记账、查看统计、管理持仓、信用账本）完全离线可用。
- 投资模块实时行情在离线状态下回退展示缓存价格（带过期提示）。

## 五、范围边界（非目标）

| 不做（MVP 阶段） | 说明 |
|----------------|------|
| 社交 / 分享 | 无多用户、无分享账单功能 |
| 自动账单导入 | 不解析银行短信 / 微信账单（规划项） |
| 交易下单 | 投资模块仅展示持仓，不提供交易功能 |
| Web / Android 版本 | 当前仅 iOS 原生 |
| 订阅 / 内购 | MVP 阶段完全免费无付费墙 |
| AI 分析 | 不集成 AI 消费建议（规划项） |

## 六、版本规划

| 里程碑 | 功能范围 | 目标 |
|--------|---------|------|
| **v0.1 MVP** | 日常账本 CRUD + 基础统计图表（月/周/年）+ 本地 SwiftData 存储 + 分类管理 | 可用的记账核心 |
| **v0.2** | 信用账本（借贷 CRUD + 还款记录 + 净额汇总）+ 应用锁（Face ID）+ iCloud 同步 | 借贷管理闭环 |
| **v0.3** | 投资理财（持仓 CRUD + 手动价格 + 平台汇总 + 配置图）+ 总览 Dashboard | 资产管理闭环 |
| **v1.0** | 实时行情 API 接入（加密货币 + 美股 + A股）+ 多币种汇率 + 本地通知提醒 | 完整功能发布 |
| **v1.x** | 自动账单导入 / AI 消费分析 / 家庭共享账本 | 规划项 |
